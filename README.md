## 🔐 What is Privacy Intelligence Engine?

**Privacy Intelligence Engine** is a comprehensive, research-grade **re-identification risk analysis framework** that goes far beyond basic anonymization checks. It implements **26+ privacy features** across multiple formal privacy models — k-Anonymity, l-Diversity, and t-Closeness — while simulating real-world attacker scenarios and providing cryptographic data protection.

Built for security researchers, data scientists, and privacy engineers who need *actionable intelligence*, not just a compliance checkbox.

## 🚀 Quick Start

### Installation

```bash
# Clone the repository
git clone https://github.com/srivarshni-k/privacy-intelligence-engine.git
cd privacy-intelligence-engine

# Create virtual environment
python -m venv venv
source venv/bin/activate        # Linux / Mac
venv\Scripts\activate           # Windows

# Install dependencies
pip install -r requirements.txt
```

### Requirements

```
pandas
numpy
```

## 📖 Usage

### Basic Analysis (Auto-detect everything)

```bash
python privacy_engine.py --data patients.csv
```

### Advanced — All Features

```bash
python privacy_engine.py --data patients.csv \
  --k 5 --l 3 \
  --attacker advanced \
  --privacy-level high \
  --show-crypto \
  --show-generalization \
  --export-json
```

### Custom Quasi-Identifiers

```bash
python privacy_engine.py --data data.csv \
  --quasi-identifiers age gender zipcode \
  --sensitive diagnosis \
  --k 10
```

### High-Security with Config File

```bash
python privacy_engine.py --data sensitive_data.csv \
  --privacy-level maximum \
  --config config.json \
  --verbose
```

## 🏛️ Module Overview

<details>
<summary><b>🎨 Visualizer</b> — Beautiful CLI rendering engine</summary>
<br/>

ANSI-colored terminal output with animated progress bars, ASCII bar charts, risk gauges, formatted tables, and boxed summaries. Every output is styled for readability and impact.

</details>

<details>
<summary><b>⚙️ PrivacyConfig</b> — Config-driven architecture</summary>
<br/>

Loads and manages all privacy parameters from a JSON file. Supports four privacy levels (`low` / `medium` / `high` / `maximum`) and full customization of k, l, t thresholds, risk weights, attacker profiles, and generalization strategies.

</details>

<details>
<summary><b>🔐 CryptoUtils</b> — Cryptographic protection layer</summary>
<br/>

SHA-256 field-level hashing with salt. Features include one-way irreversible protection, collision resistance demonstration, avalanche effect showcase, and full dataset integrity verification via hash fingerprinting.

</details>

<details>
<summary><b>🔍 PrivacyAnalyzer</b> — Core privacy analysis engine</summary>
<br/>

Implements all three formal privacy models: k-Anonymity (equivalence class grouping + violation detection), l-Diversity (sensitive attribute diversity checking), and t-Closeness (Earth Mover's Distance-based distribution analysis). Also handles uniqueness scoring, adaptive risk calculation, and human-readable warning generation.

</details>

<details>
<summary><b>⚔️ AttackSimulator</b> — Re-identification attack engine</summary>
<br/>

Simulates three attacker profiles — **Casual** (knows 1 attribute, public records), **Insider** (partial knowledge), and **Advanced** (all quasi-identifiers). Also simulates a **Linkage Attack** by generating a synthetic external dataset and measuring successful record joins.

</details>

<details>
<summary><b>🛡️ PrivacyMitigation</b> — Protection strategy engine</summary>
<br/>

Implements automatic generalization (age ranges, ZIP prefix truncation), suppression recommendations with column-level uniqueness analysis, and a privacy budget simulator that models the privacy-utility tradeoff across all four privacy levels.

</details>

<details>
<summary><b>📄 ReportGenerator</b> — Output and visualization</summary>
<br/>

Generates comprehensive human-readable TXT reports covering every analysis stage, exports full structured JSON results, and renders the CLI visualization dashboard with bar charts, risk gauges, metrics tables, and attack comparison charts.

</details>

## 🔬 Privacy Models Explained

### k-Anonymity
Each record is indistinguishable from at least **k-1** other records based on quasi-identifiers. Groups with fewer than k records are flagged as violations.

### l-Diversity
Each equivalence class must contain at least **l** distinct values of the sensitive attribute — preventing attribute disclosure even when k-anonymity holds.

### t-Closeness
The distribution of sensitive attribute values within each group must be within distance **t** of the overall dataset distribution, measured via Earth Mover's Distance.

---

## ⚔️ Attacker Profiles

```
CASUAL    →  Knows 1 attribute    (public records, social media)
INSIDER   →  Knows partial attrs  (employee, researcher)
ADVANCED  →  Knows all attrs      (sophisticated adversary, nation-state)
```

Each profile calculates per-record re-identification probability and reports average/median/max success rates and guaranteed re-identification counts.

## 🐛 Troubleshooting

| Issue | Cause | Fix |
|:---|:---|:---|
| `ModuleNotFoundError` | Missing packages | `pip install pandas numpy` |
| `KeyError` on column | Wrong column name | Use `--quasi-identifiers` explicitly |
| Empty bar charts | No data in that category | Expected — all records are low-risk |
| Config not loading | Invalid JSON | Validate with `python -m json.tool config.json` |
| Generalization no-op | Columns `age`/`zipcode` not present | Rename columns or extend strategy in config |
